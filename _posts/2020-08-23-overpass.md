Through simple guess, we find no `robots.txt` and do find an admin login at `/admin`. Checking the source, there are 3 javascript files: cookie.js, login.js, and main.js.

**login.js**:
```
...
async function login() {
    const usernameBox = document.querySelector("#username");
    const passwordBox = document.querySelector("#password");
    const loginStatus = document.querySelector("#loginStatus");
    loginStatus.textContent = ""
    const creds = { username: usernameBox.value, password: passwordBox.value }
    const response = await postData("/api/login", creds)
    const statusOrCookie = await response.text()
    if (statusOrCookie === "Incorrect credentials") {
        loginStatus.textContent = "Incorrect Credentials"
        passwordBox.value=""
    } else {
        Cookies.set("SessionToken",statusOrCookie)
        window.location = "/admin"
    }
}
```
If `statusOrCookie != "Incorrect credentials"`, a SessionToken cookie is set to the value. So if we set the cookie ourselves to a success status (or potentially anything since the cookie isn't being validated), we might be allowed through.

Run `Cookies.set("SessionToken", 400)` in the dev Console and refresh the page.
